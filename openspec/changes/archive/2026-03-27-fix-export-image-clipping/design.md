## Context

当前 PNG 导出由 `export-service.ts` 生成渲染场景并输出位图。现有流程虽然会根据内容范围估算尺寸，但在节点非常多（例如同层级 100 个）时，用户反馈导出结果仍可能出现可见截断，表现为只显示部分节点。该问题属于导出能力的正确性问题，且位于 renderer/main 跨边界链路（`ExportDialog -> IPC -> export-service`）中，需以规范化边界计算与测试回归方式修复。

约束：
- 保持现有 PNG 比例输入（50%-400%）与 API 兼容。
- 不改变当前布局算法语义，仅修正导出场景的边界与尺寸策略。
- 在极端大图场景下仍需满足位图安全上限，避免 Electron/内存崩溃。

## Goals / Non-Goals

**Goals:**
- 基于导出场景的真实几何边界（最上/最左/最下/最右）计算内容框，确保所有可见节点与连线都被包含。
- 导出尺寸计算顺序固定为：先“完整内容入框”，再按用户导出比例计算最终像素尺寸。
- 在达到栅格安全上限时仅做全局等比缩放，保证“全部内容仍在图内”，禁止局部裁切。
- 为高节点密度场景补充可回归的自动化测试。

**Non-Goals:**
- 不重写 `computeLayout` 布局算法。
- 不改动导出 UI 交互与比例输入范围。
- 不在本次变更中引入分片导出、多页导出或流式写图能力。

## Decisions

### Decision 1: 显式抽象导出内容边界（Content Bounds）
- 方案：在 `buildRenderScene` 内部统一计算并持有 `top/left/bottom/right`，边界来源包括：
  - 节点矩形（x, y, x+width, y+height）
  - 连线路径控制点与端点（start/end/cp1/cp2）
- 理由：将边界作为一等数据，避免“尺寸推导散落在多个步骤”导致边界遗漏。
- 备选方案：继续沿用隐式 `minX/maxX` 局部变量。未选原因：可读性差，不利于后续验证与测试断言。

### Decision 2: 导出尺寸采用“边界优先 + 最小尺寸兜底”
- 方案：逻辑尺寸 `logicalWidth/Height = max(contentSpan + 2*padding, minWidth/minHeight)`。
- 理由：保证节点多时会扩张画布，不会因固定基准尺寸导致边缘内容不可见。
- 备选方案：固定最小画布后仅缩放内容。未选原因：会退化为“节点多就变小”，不满足问题诉求。

### Decision 3: 比例应用在完整边界之后
- 方案：先完成逻辑场景平移（把 `left/top` 映射到 padding），再把用户比例统一作用于坐标、尺寸、字体、连线宽度。
- 理由：保证比例语义是“分辨率倍率”，而不是“裁切窗口倍率”。
- 备选方案：先缩放再估算边界。未选原因：边界值会受舍入误差放大，容易出现 1-2px 裁切并增加复杂度。

### Decision 4: 极端尺寸下仅允许全局等比安全收敛，不允许裁切
- 方案：保留安全上限控制，但强制使用同一 `fitScale` 同步缩放画布与全部几何元素，确保四向边界仍完整落在图内。
- 理由：兼顾“完整导出”与运行时稳定性（内存/最大纹理尺寸约束）。
- 备选方案：超限即报错。未选原因：用户体验差，且不能满足“尽可能导出全部节点”。

### Decision 5: 增加“高同层节点”场景测试
- 方案：在 `tests/export/export-service.test.ts` 新增大同层节点用例（例如 100 siblings），断言导出图尺寸随边界增长且右下区域存在非背景像素。
- 理由：直接覆盖本次用户问题场景，避免回归。
- 备选方案：只做人工验证。未选原因：回归成本高，稳定性不足。

## Risks / Trade-offs

- [Risk] 超大导图导致导出耗时上升或内存压力增加  
  → Mitigation: 维持安全上限与统一 `fitScale` 收敛，测试覆盖极端尺寸。

- [Risk] 四向边界扩张后默认导出分辨率显著增大，文件体积上升  
  → Mitigation: 保留现有比例输入；100% 作为默认，用户可主动降低比例。

- [Risk] 连线控制点纳入边界后会引入额外留白  
  → Mitigation: 通过统一 padding 策略控制视觉留白，并确保不裁线。

## Migration Plan

1. 在 `export-service.ts` 重构边界计算与尺寸推导顺序（边界优先、比例后置）。
2. 对 PNG 渲染流程进行回归，确保 SVG/PNG 边界一致。
3. 增加导出服务测试：高同层节点完整性、边界点覆盖、极端尺寸安全收敛。
4. 运行现有导出与 IPC 测试集，确认接口兼容。
