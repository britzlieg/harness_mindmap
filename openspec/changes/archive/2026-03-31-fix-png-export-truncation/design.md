标题: PNG 导出截断问题 — 设计
概述
- 解决在当前项目中导出 PNG 时出现的截断问题，根因分析将指导实现。

设计目标
- 确保所有画布在导出时完整数据写入，包括大分辨率和离屏渲染。
- 使导出管线对流式/缓冲边界以及库的差异具备鲁棒性。
- 增加能够复现截断场景的目标测试。

实现思路
- 复现：创建高分辨率画布和离屏渲染的测试场景以复现实验。
- 根因假设：可能包括提前关闭流、分块写入对齐问题、或 PNG 编码路径中的缓冲未完全刷新。
- 修复计划:
  1. 检查 PNG 导出路径（mindmap-app/electron/services/export.* 或等效实现）以定位写入器/编码器。
  2. 替换或封装 PNG 写入逻辑，使用确定性的刷新策略：确保所有块在关闭流前写入完成。
  3. 将块边界与编码器输出对齐，Finish 时确保对压缩缓冲的完全刷新。
  4. 为大画布和离屏场景添加回归测试。
  5. 将变更控制在低风险范围，并尽量降低性能影响。

非功能性考量
- 兼容性：不变更 API 表面，仅修改内部导出行为。
- 性能：额外的刷新调用应尽量不显著增加导出时间。
- 安全性：无外部输入，错误路径处理要安全且不暴露敏感信息。

风险与缓解
- 风险：PNG 编码的回归。缓解：增加测试并运行 CI。
- 风险：因缓冲增加导致的内存占用上升。缓解：尽量使用流式处理并限制缓冲。

接口影响
- 受影响：PNG 导出服务模块及其测试。
- 未来可在文档中补充导出行为的说明。
