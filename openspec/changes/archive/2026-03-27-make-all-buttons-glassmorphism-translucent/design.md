## Context

当前应用已经有部分按钮采用毛玻璃风格，但覆盖范围不完整，部分按钮仍是偏实色或透明度不足，导致同一界面内按钮质感不一致。  
该变更涉及全局按钮样式类、组件局部覆盖样式、按钮状态样式与回归测试，属于跨组件视觉规范收敛改造。

## Goals / Non-Goals

**Goals:**

- 统一所有按钮为半透明毛玻璃风格（包含工具栏、侧栏、弹窗、画布控制等区域）。
- 明确定义按钮在 `hover/active/focus-visible/disabled` 各状态下的视觉规则并保持一致。
- 通过样式 token 保证后续新增按钮默认继承该规范。

**Non-Goals:**

- 不调整按钮的业务行为、点击逻辑与快捷键。
- 不重做布局结构或组件层级。
- 不引入新的 UI 框架或设计库。

## Decisions

### 决策 1：以统一按钮基类 token 化实现，而非逐组件手工覆盖

- 方案：将毛玻璃参数（透明度、模糊、边框、高光、阴影）集中到统一按钮样式体系中，按钮变体仅做轻量差异。
- 原因：避免多个组件出现重复样式与后续漂移。
- 备选：
  - 每个组件单独改样式：短期快，但长期难维护且难保证一致性。

### 决策 2：强制所有按钮背景半透明，并引入模糊层

- 方案：按钮背景统一使用带 alpha 的颜色，搭配 `backdrop-filter` 实现毛玻璃；不允许纯不透明实色背景作为常规按钮样式。
- 原因：满足“所有按钮半透明 + 毛玻璃”的核心目标。
- 备选：
  - 仅主按钮半透明：风格不统一，不满足需求。

### 决策 3：状态样式统一规范化

- 方案：定义全局状态策略：
  - `hover`: 提升边框与背景亮度；
  - `active`: 轻微下压与阴影收敛；
  - `focus-visible`: 明确焦点环；
  - `disabled`: 保持半透明但降低对比与交互反馈。
- 原因：保证可用性与一致交互反馈。

## Risks / Trade-offs

- [风险] 全量按钮统一毛玻璃后，个别场景层级对比不足。  
  -> 缓解：为文字和边框设置最小对比阈值，并保留按变体微调 token。

- [风险] 部分系统/环境对 `backdrop-filter` 支持有限。  
  -> 缓解：保留无模糊回退样式（半透明背景 + 边框 + 阴影）。

- [风险] 过多阴影或高光可能产生视觉噪声。  
  -> 缓解：约束阴影强度与透明度范围，统一从 token 管理。

## Migration Plan

1. 先统一 `ui-button` 基类与变体样式 token。  
2. 排查并替换组件中的局部按钮覆盖样式。  
3. 执行样式回归测试与关键页面手动检查。  
4. 如出现兼容问题，回退到半透明无模糊方案。

## Open Questions

- 是否为“危险按钮/警示按钮”保留更高强调度的专用变体？  
- 是否需要在后续版本中增加用户可选“低模糊模式”？

## Implementation Notes

- 已新增统一按钮毛玻璃 token（背景、边框、阴影、状态）并注入主题运行时变量。
- 全量按钮类型（`ui-button`、`app-sidebar-tab`、`ui-option`、`ui-theme-card`、`ui-format-option`）均已统一为半透明毛玻璃表现。
- 主按钮与次按钮均避免不透明实色背景，状态变化在保持半透明前提下进行。
- 已补充并通过按钮相关回归测试，覆盖工具栏、侧栏、弹窗等关键区域。
