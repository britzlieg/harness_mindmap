# PNG 导出修复 - 边缘情况与后续建议

## 已修复的问题

### 1. 内容边界计算
- **问题**: 当导图包含大量节点时，边界计算可能未正确覆盖所有内容
- **修复**: 确保 `fitSceneForRaster` 使用四极值点（最左、最上、最右、最下）计算边界，并在缩放后重新验证

### 2. Electron 单帧捕获失败时未降级
- **问题**: 单帧捕获验证失败后直接返回，未尝试分块捕获
- **修复**: 添加注释说明，当单帧捕获尺寸验证失败时 fall through 到分块捕获逻辑

### 3. 分块捕获缺少边界检查
- **问题**: `copyBgraTileToRgba` 缺少目标缓冲区边界检查
- **修复**: 添加边界验证、尺寸夹紧和警告日志

### 4. PNG 验证不够完善
- **问题**: 只验证尺寸，未验证内容完整性
- **修复**: 在 `renderPng` 中增加三级验证链（尺寸验证 → 内容完整性验证 → fallback）

## 已知边缘情况

### 1. 超大导图导出（接近 12000px 限制）
- **现象**: 当导图非常大时，会自动缩小到 `MAX_RASTER_EXPORT_EDGE` (12000px) 限制内
- **当前行为**: 等比缩放场景以适应限制，保持内容完整但可能降低清晰度
- **建议**: 在 UI 中增加导出前尺寸提示，告知用户预期输出尺寸

### 2. 分块捕获的平台差异
- **现象**: Windows/macOS/Linux 上的 BrowserWindow 渲染行为可能有细微差异
- **当前行为**: 使用分块捕获 + 拼接，有边界检查和警告日志
- **建议**: 增加跨平台视觉回归测试，使用截图对比验证

### 3. 软件 fallback 渲染质量
- **现象**: 当 Electron 捕获失败时，fallback 到软件渲染 (`renderExportSceneToPngFallback`)
- **当前行为**: 软件渲染使用简化的文字渲染（伪字模），质量低于 Electron 原生渲染
- **建议**: 考虑使用 Canvas API 或引入外部库（如 sharp）作为更高质量的 fallback

### 4. 极端缩放比例
- **现象**: 用户可能输入极端缩放值（如 50% 或 400%）
- **当前行为**: 限制在 50%-400% 范围内，超出范围会被拒绝
- **建议**: 在 UI 中增加常用缩放比例快捷选项（100%、200%、300%）

### 5. 空节点或稀疏导图
- **现象**: 导图只包含少量节点或大量空白节点
- **当前行为**: 使用 `FALLBACK_EXPORT_EDGE` (240px) 作为最小尺寸
- **建议**: 当前行为合理，无需修改

## 后续优化建议

### 短期（1-2 周）
1. **增加导出预览功能**
   - 在保存前显示导出预览
   - 允许用户调整缩放比例并实时预览效果
   
2. **改进错误提示**
   - 当导出失败时，提供更具体的错误信息（如"内容超出最大尺寸限制"）
   - 建议用户降低缩放比例或简化导图

3. **性能优化**
   - 对于大导图，考虑使用 Web Worker 进行后台渲染
   - 增加导出进度指示器

### 中期（1-2 月）
1. **引入高质量 fallback 渲染器**
   - 考虑使用 `sharp` 库进行 PNG 编码
   - 或使用 Canvas API 配合 node-canvas

2. **支持更多导出格式**
   - JPEG（适合包含大量图像的导图）
   - WebP（更小的文件大小）
   - PDF（适合打印）

3. **批量导出功能**
   - 允许用户一次性导出多个缩放比例的 PNG
   - 支持导出为 ZIP 压缩包

### 长期（3-6 月）
1. **云端导出服务**
   - 将渲染密集型任务卸载到云端
   - 支持超大规模导图导出（>12000px）

2. **增量导出**
   - 只重新渲染变化的部分
   - 缓存已渲染的节点和连线
