## Why

当前主题色下菜单与侧栏文字颜色在部分主题中对比度不足，出现刺眼、发灰或难以辨认的问题，直接影响阅读效率与可用性。  
需要基于现有 UI 色轮设计重新选择文字色系，保证在不同主题下都能稳定可读。

## What Changes

- 重新定义菜单与侧栏文字颜色映射规则，优先使用与主题背景有明确对比关系的色轮相邻/互补色。
- 区分主文本、次级文本、禁用文本与强调文本的色阶，避免单一亮色在深浅背景中通用造成刺眼。
- 增加主题下字体可读性约束（对比度阈值与视觉舒适度要求），覆盖默认主题与内置主题集合。
- 补充菜单与侧栏在不同主题下的样式回归测试，防止后续主题扩展引入低可读配色。

## Capabilities

### New Capabilities

- （无）

### Modified Capabilities

- `mindmap-theme-management`: 扩展主题系统中的字体色映射策略，要求主题色下菜单与侧栏文字满足可读性规则。
- `mindmap-ui-visual-polish`: 调整菜单与侧栏文本层级配色规范，统一主/次/禁用/强调文本视觉层次。

## Impact

- 前端样式层：`CSS variables`、菜单与侧栏组件文本色 token。
- 主题系统：主题 token 解析与 UI 色轮映射逻辑。
- 测试：主题回归测试、视觉相关快照或 DOM 样式断言。
