## 为什么

用户虽然可以保存思维导图，但目前无法在应用内稳定地重新打开已有 `.mindmap` 文件，跨会话连续编辑体验被打断。与此同时，桌面窗口仍显示英文顶层菜单（`File`、`Edit` 等），且 PNG 导出会出现缺少连线和文字的问题，导致导出结果不可用。

## 变更内容

- 增加明确的“打开思维导图”文件操作，允许用户选择并加载已有 `.mindmap` 文件到当前会话。
- 调整桌面端菜单行为，隐藏英文顶层原生应用菜单。
- 修复 PNG 导出渲染，确保导出图像包含可视思维导图中的节点文字与连线。
- 保证打开/导出失败时的编辑安全性（不崩溃、错误可处理、不静默破坏数据）。

## 能力范围

### 新增能力
- 无。

### 修改能力
- `mindmap-file-operations`：补充“打开已有 `.mindmap` 文件”的需求。
- `menu-chinese-localization`：补充“应用窗口隐藏 Electron 英文顶层菜单”的需求。
- `mindmap-png-export`：强化 PNG 导出要求，确保导出结果包含完整可视内容（文字 + 连线），而非空白或残缺图像。

## 影响

- 影响模块：Electron 主进程菜单初始化、文件打开 IPC 流程、思维导图文件加载逻辑、PNG 导出渲染链路。
- 测试影响：需要补充/更新文件操作集成测试、菜单本地化断言测试、PNG 导出内容校验测试。
- 不涉及外部 API 变更；本次为桌面端体验与导出可靠性改进。
